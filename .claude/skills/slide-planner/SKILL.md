# Slide Planner

大量の情報や素材を受け取り、対話を通じてプレゼンの骨格を設計する。
最終的に `public/slide-plan.md` を出力し、`/slide-generator` への入力として使える状態にする。

## 重要な原則

- **情報を全部スライドに入れようとしない**。プレゼンは「伝える」ための道具であり「載せる」ための道具ではない
- **視聴者の視点**で考える。発表者が言いたいことではなく、聞き手が知りたいことを優先
- **1スライド1メッセージ**。2つ以上のことを伝えようとするスライドは分割する

## 対話フロー

### Step 1: 情報の受け取りと整理

ユーザーから提供された情報（$ARGUMENTS、ファイル、URL等）を読み込み、以下を抽出する：

- **事実・データ**: 数値、実績、結果
- **ストーリー**: 経緯、背景、文脈
- **主張・提案**: 結論、お願い、ネクストアクション
- **補足資料**: 参考情報、詳細データ

整理した情報を箇条書きで提示し、漏れがないか確認する。

### Step 2: 核心を引き出す（必ず質問する）

以下を**ユーザーに質問**して明確にする。推測で進めない。

1. **誰に向けて？** → 視聴者は誰か（経営層、エンジニア、クライアント、一般等）
2. **何を持ち帰ってほしい？** → プレゼン後に聞き手に起こしてほしいアクションは？
3. **一言で言うと？** → このプレゼン全体を一文で要約すると？
4. **制約は？** → 持ち時間、ページ数の希望、必須で入れたい内容
5. **トーンは？** → フォーマル/カジュアル、データ重視/ストーリー重視

### Step 3: 情報の取捨選択

Step 1 の情報を Step 2 の回答に照らして仕分ける：

- **🟢 スライドに入れる**: 核心メッセージの裏付けになる情報
- **🟡 付録・補足**: あると良いが本編に入れると冗長な情報
- **🔴 今回は使わない**: テーマから外れる、詳細すぎる、別の機会に回す

仕分け結果を提示して確認する。

### Step 4: 構成設計

ページ数と構成を設計する。目安：

| 持ち時間 | ページ数 | 構成 |
|---|---|---|
| 5分 | 5〜8枚 | タイトル→要点3つ→まとめ |
| 10分 | 8〜15枚 | タイトル→目次→本編3セクション→まとめ |
| 20分 | 15〜25枚 | タイトル→目次→本編4-5セクション→まとめ→Q&A |
| 30分+ | 25〜40枚 | 上記＋詳細データ・事例 |

各スライドについて：
- **ページ番号**
- **レイアウト番号と名前**（[reference.md](../slide-generator/reference.md) から選択）
- **このスライドで伝えること**（1文）
- **デコレーション**: データ系→lines / 感情系→corners
- **キーワード・データ**: スライドに入れる具体的な値

構成案を提示して確認する。

### Step 5: MDファイル出力

確定した構成を `public/slide-plan.md` に出力する：

```markdown
# {プレゼンタイトル}

## メタ情報
- 視聴者: {対象}
- 持ち時間: {時間}
- トーン: {フォーマル/カジュアル}
- 一言要約: {核心メッセージ}
- フォント: en={英文見出し}/{英文本文} / ja={日本語見出し}/{日本語本文}

## スライド構成

### Slide 1 — タイトル
- layout: 1 (title)
- decoration: corners [top-left, bottom-right]
- title: {タイトル}
- subtitle: {サブタイトル}

### Slide 2 — 目次
- layout: 4 (table-of-contents)
- decoration: corners [top-right, bottom-left]
- items:
  - 01: {セクション1名}
  - 02: {セクション2名}
  - 03: {セクション3名}

### Slide 3 — {このスライドの目的}
- layout: {番号} ({ID})
- decoration: {lines-double / corners [...] / none}
- content:
  - title: {見出し}
  - body: {本文の要約}
  - image: {必要なら画像の説明}

...

## 使わなかった情報（参考）
- {今回省いた情報を記録しておく}
```

## 次のステップの案内

MDファイル出力後、以下を案内する：

1. `slide-plan.md` の内容を確認・調整
2. `/slide-generator` に slide-plan.md の内容を渡して deck.json を生成
3. `/validate-deck` で検証
4. `/generate-image-prompts` で画像生成依頼を出力
